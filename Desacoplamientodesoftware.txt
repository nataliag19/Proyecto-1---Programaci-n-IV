\section*{Introducción}
Se desarrolló el sistema NutriUTP, cuyo propósito es simular el funcionamiento de alguna de las cafeterías de la Universidad Tecnológica de Pereira, permitiendo administrar un menú diario, ofrecer opciones como platos vegetarianos y calcular el precio final de los alimentos según el perfil del estudiante.

Más allá de la funcionalidad, el sistema fue diseñado buscando que sea claro, ordenado y fácil de mantener. Esto significa que cada parte del programa cumple una función específica, evitando mezclar responsabilidades y facilitando futuras modificaciones, como cambios en los subsidios o en la forma de gestionar los platos.De esta manera, el proyecto aplica buenas prácticas de programación que permiten construir soluciones más organizadas y escalables.

\section*{Clase Comensal}
Esta clase representa a la persona que va a comprar el almuerzo. Cada vez que se crea un nuevo comensal:

\begin{itemize}
    \item Se le asigna automáticamente un identificador único.
    \item Se guarda el tipo de subsidio que tiene (alto, medio, bajo o ninguno).
\end{itemize}

Además, esta clase se encarga de calcular el precio final del plato dependiendo del subsidio:

\begin{itemize}
    \item Subsidio alto → paga el 50%
    \item Subsidio medio → paga el 70%
    \item Subsidio bajo → paga el 90%
    \item Sin subsidio → paga el valor completo
\end{itemize}

Esto permite que toda la lógica del descuento esté en un solo lugar.

\begin{lstlisting}
class Comensal:
    
    _contador_comensal = 0

    MAPEO_SUBSIDIOS = {
        "alto": 0.5,
        "medio": 0.7, 
        "bajo": 0.9,  
        "ninguno": 1.0
    }

    def __init__(self, tipo_subsidio = "ninguno"):
        Comensal._contador_comensal += 1
        self.id_estudiante = Comensal._contador_comensal
        self.tipo_subsidio = tipo_subsidio

    def calcular_descuento(self, precio):
        multiplicador = self.MAPEO_SUBSIDIOS.get(self.tipo_subsidio, 1.0)
        return precio * multiplicador
\end{lstlisting}

\section*{Clase MenuDia}

Esta clase representa el menú disponible en un día específico. Sus funciones principales son:

\begin{itemize}
    \item Guardar la fecha del menú
    \item Almacenar los platos disponibles
    \item Permitir agregar nuevos platos
\end{itemize}

También tiene un método importante que permite filtrar los platos según la preferencia del usuario:

\begin{itemize}
    \item Opción estándar
    \item Opción vegetariana
\end{itemize}

Para esto utiliza un método interno llamado -aislar-seleccion, que se encarga de hacer la búsqueda sin exponer cómo se guardan los datos internamente. Esto ayuda a que el sistema sea más ordenado y fácil de modificar.

\begin{lstlisting}
from datetime import date
class MenuDia:
    def __init__(self, fecha = date.today()):
        self.fecha = fecha
        self._opciones = []

    def agregar_plato(self, plato):
        self._opciones.append(plato)

    def mostrar(self):
        print(f"\n=== MENÚ ({self.fecha}) ===")
        for iterador, plato in enumerate(self._opciones):
            print(f"{iterador+1}. {plato.descripcion_detallada()}")

    def seleccionar_opcion(self, tipo_preferencia):
        return self._aislar_seleccion(tipo_preferencia)
    
    def _aislar_seleccion(self, tipo_preferencia):
        es_veg = (tipo_preferencia == 2)
        platos_filtrados = [plato for plato in self._opciones if plato.es_vegetariano == es_veg]
        return {str(i+1): plato for i, plato in enumerate(platos_filtrados)}
\end{lstlisting}

\section*{Clase Plato}

Esta clase representa cada uno de los platos disponibles en el menú. Cada plato tiene:

\begin{itemize}
    \item Nombre
    \item Precio base
    \item Si es vegetariano o no
    \item Cantidad disponible
\end{itemize}

También incluye funciones para:

\begin{itemize}
    \item Consultar su precio
    \item Verificar si está disponible
    \item Reducir la cantidad cuando se vende
    \item Mostrar una descripción completa del plato
\end{itemize}

Esto permite manejar cada plato de manera independiente.

\begin{lstlisting}
class Plato:
    def __init__(self, nombre = "Plato Default", precio = 1000, veg = False, cantidad = 1):
        self.nombre = nombre
        self._precio_base = precio
        self.es_vegetariano = veg
        self.cantidad = cantidad

    def precio(self):
        return self._precio_base

    def esta_disponible(self):
        return self.cantidad > 0

    def reducir_stock(self):
        if self.cantidad > 0:
            self.cantidad -= 1

    def descripcion_detallada(self):
        tipo = "Vegetariano" if self.es_vegetariano else "Normal"
        estado = "Disponible" if self.esta_disponible() else "Agotado"
        return f"{self.nombre} | {tipo} | ${self._precio_base} | {estado} | Stock: {self.cantidad}"
\end{lstlisting}

\section*{Clase ProcesadorVenta}

Esta clase es la encargada de gestionar las compras. Cuando un usuario selecciona un plato:

\begin{itemize}
    \item Verifica si el plato está disponible
    \item Obtiene el precio base
    \item Aplica el descuento del comensal
    \item Reduce el stock del plato
    \item Registra la venta
    \item Suma el valor al total recaudado
\end{itemize}

Además, guarda un historial de todas las ventas realizadas, lo que permite generar reportes posteriormente.

\begin{lstlisting}
class ProcesadorVenta:
    def __init__(self):
        self._ventas = []
        self._total = 0

    def generar_tiquete(self, comensal, plato):
        if not plato.esta_disponible():
            return None

        precio_base = plato.precio()
        precio_final = comensal.calcular_descuento(precio_base)

        plato.reducir_stock()

        venta = {
            "cliente": comensal.id_estudiante,
            "plato": plato.nombre,
            "precio": precio_final
        }

        self._ventas.append(venta)
        self._total += precio_final

        return venta

    def validar_pago(self):
        return self._ventas
\end{lstlisting}

\section*{Características importantes del diseño}
Este sistema fue construido pensando en que sea fácil de entender y modificar. Algunas características importantes son:

\begin{itemize}
    \item Cada clase tiene una función específica
    \item La información está organizada
    \item Los cálculos (como descuentos) están separados del resto
    \item Es fácil agregar nuevos tipos de platos o cambiar reglas

\end{itemize}

\section*{Conclusión}

El desarrollo del sistema NutriUTP permitió aplicar de manera práctica conceptos fundamentales de diseño de software, evidenciando la importancia de estructurar correctamente un programa más allá de su funcionalidad básica. A lo largo del proyecto, se logró construir una solución modular en la que cada componente cumple una responsabilidad específica, facilitando la organización del código y su posterior mantenimiento.

Uno de los principales aprendizajes fue la implementación de los mecanismos de encapsulamiento que protegen la información interna de las clases. Estos ayudaron a una separación clara entre la lógica de negocio y la interacción con el usuario. Esta división permite que cambios en reglas como los subsidios, la gestión del menú o el procesamiento de ventas puedan realizarse sin afectar directamente la interfaz, lo que mejora la flexibilidad del sistema.

Finalmente, este proyecto demuestra que un buen diseño no solo facilita el cumplimiento de los requisitos actuales, sino que también prepara el sistema para posibles extensiones, como la incorporación de nuevos tipos de usuarios, reglas de negocio o funcionalidades adicionales.
